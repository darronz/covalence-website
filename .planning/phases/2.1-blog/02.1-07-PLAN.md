<objective>
Fix a latent content bug in `src/content/docs/docs/getting-started.md`: 4 fenced code blocks are indented 4 spaces inside `<TabItem>` JSX, which CommonMark interprets as indented code blocks rather than fenced ones. Result: triple-backticks leak as literal text, language tags are lost, and `astro-expressive-code` cannot apply syntax colouring even though it wraps them in a container.

**This is a pre-existing issue on production covalence.app (identical source in commit `b1bfd58`).** Wave 5's "Surface E regression" was misattributed — the H5 fix in Plan 02.1-06 eliminated a real theme-selector mismatch but didn't change this page visually because the indented-fence issue dominates the symptom. Extending Phase 2.1 gap closure to fix the content bug sends a genuinely greener `/docs/*` to production.

Output: After this plan ships on `gsd/phase-2.1-blog`, `/docs/getting-started/` renders the four per-client code blocks with syntax colouring (Starlight's default theme), copy buttons, and frame chrome — the same treatment other docs pages' non-indented fences already receive. The Connect-Your-AI-Client section becomes five linear H3 sub-sections (Claude Desktop / Claude Code / Cursor / OpenCode / ChatGPT) with all five clients visible at once — no hidden-behind-click content. The `import { Tabs, TabItem }` line that had been rendering as literal prose at the top of the page is removed.

**Revision note (2026-04-22):** An earlier draft of this plan used Starlight's `<Code>` component inside `<Tabs>`. That approach failed at execution: this project has no `@astrojs/mdx` integration and the file is a plain `.md`, so JSX and ES-module syntax render as literal text. Investigation also revealed the pre-existing `<Tabs>` had never rendered as interactive tabs (zero `role="tablist"` in the HTML) — they were passing through as lowercase HTML tags with all client content visible linearly. Option B (this plan) drops the broken JSX entirely; Option A (install `@astrojs/mdx` to make Tabs work) was considered and rejected as larger scope for no net UX gain over five H3 sub-sections.
</objective>
