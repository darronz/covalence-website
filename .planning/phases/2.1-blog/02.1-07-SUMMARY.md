## Accomplishments

- **Surface E content bug closed.** The 4 previously-indented fenced code blocks inside `<TabItem>` were being parsed as CommonMark indented-code-blocks — triple-backticks leaked as literal text, language tags were lost, and Expressive Code could not wrap them. Post-fix: 0 leaked fences, 4 `expressive-code` wrappers, `ec-line` count doubled from 8 to 16 (the content now renders as full-width EC-lines instead of partial indented-code interspersed with leaked fences).
- **Stray `import` line removed.** The `import { Tabs, TabItem } from '@astrojs/starlight/components';` at line 6 had been rendering as literal prose at the top of the page (no @astrojs/mdx integration in this project). That line is gone from both source and rendered HTML.
- **All five clients now visible linearly.** The pre-existing `<Tabs>` had never rendered as interactive tabs (0 `role="tablist"` in HTML on both preview and production) — they were passing through as lowercase HTML with all TabItem content rendered one after another. The rewrite makes that linear-all-visible intent the actual design: five H3s with anchor IDs (`#claude-desktop`, `#claude-code`, `#cursor`, `#opencode`, `#chatgpt`), all scannable without a click.
- **Wave 4 + Wave 6 invariants preserved.** RSS alternate link still present at count=1 on `/docs/getting-started/` and count=1 on `/posts/index.html`. Sibling /docs/* pages (`ai-instruction`, `mcp-tools`, `spaces`, `core-memories`, `keyboard-shortcuts`) all have `ec-line` counts identical to pre-fix — no collateral damage.
- **ChatGPT section prose preserved verbatim.** The user has a pending todo (`deb117b`, `b9c86b2`) to fix the MCP-support claim; this plan deliberately did NOT pre-empt it.
- **Branch pushed.** `7ff888e..f37c81a gsd/phase-2.1-blog -> gsd/phase-2.1-blog` — 5 commits pushed in one fast-forward (4 prior unpushed local commits + Task 2's fix). CF Pages will auto-rebuild the preview.
