# External Integrations

**Analysis Date:** 2026-04-18

This is a pure static marketing + docs site built with Astro + Starlight. It has **no backend, no runtime API calls, no SDK clients, no auth, and no database**. Most categories below are intentionally "None" — documented explicitly so future contributors don't go looking for integrations that don't exist.

## APIs & External Services

**Runtime API clients:** None. The site is fully static; no fetch/axios/SDK code exists in `src/`.

**Build-time data sources:**
- `src/data/latest-release.json` — Read at build time by `src/components/Hero.astro` (via `import latestRelease from '../data/latest-release.json'`) to render the Covalence app version and download size in the hero section.
- `src/data/releases.json` — Historical release archive (seeded with v1.3.2). Not currently imported by any component but is the append-only target for the `new-release` workflow.

**Outbound links to external origins (rendered HTML only, not code-level integrations):**
- `https://covalence.app/releases/Covalence-latest.dmg` and `https://covalence.app/releases/Covalence-<version>.dmg` — Download links in `src/components/Hero.astro`, `src/components/Nav.astro`, `src/components/Footer.astro`, `src/content/docs/docs/index.md`, `src/content/docs/docs/getting-started.md`, and the release JSON files. These DMGs are hosted at the same domain but served from a path (`/releases/*`) that `public/_routes.json` explicitly excludes from the static Pages bundle — meaning Cloudflare Pages forwards those requests to a separate origin (CDN / R2 / Pages Function), not resolved by Astro.
- `/appcast.xml` — Also excluded via `public/_routes.json`; served by the same upstream as `/releases/*`. No code in this repo generates it.

## Data Storage

**Databases:** None. The Covalence macOS app itself uses local SQLite + sqlite-vec (documented in `src/components/Architecture.astro` and `src/content/docs/docs/*.md`), but that is the product being marketed, not a dependency of this website.

**File Storage:**
- Release binaries (DMGs) and `appcast.xml` are served from `https://covalence.app/releases/*` via a non-Astro upstream (see `public/_routes.json`). The mechanism (R2, Pages Functions, external CDN) is not defined in this repo.
- Everything else is static files committed under `public/` (fonts, icons, OG image) or generated by `astro build` into `dist/` (HTML, `_assets/`, `pagefind/`, sitemap).

**Caching:** None at the application level. Any caching is whatever the static host (Cloudflare Pages) applies to the emitted assets.

## Authentication & Identity

None. There is no login, no sessions, no user accounts, no gated content. The product itself explicitly advertises "No accounts. No telemetry. No cloud." (`src/components/Architecture.astro`).

## Monitoring & Observability

**Error Tracking:** None. No Sentry, Bugsnag, Rollbar, or similar SDK.

**Analytics:** None detected. No Google Analytics, Plausible, Fathom, PostHog, or equivalent snippet in `src/layouts/Base.astro` or anywhere else.

**Logs:** Not applicable (no server runtime).

## CI/CD & Deployment

**Hosting:** Cloudflare Pages.
- Evidence: `public/_routes.json` (Pages-specific routing config), commit `63401d6` ("pin Node version for CF Pages build determinism"), and `.nvmrc` pinning Node `22.16.0` for reproducible Pages builds.
- Pages build command is presumably `npm run build` (producing `dist/`); no Pages config file is committed in this repo, so the build settings live in the Cloudflare Pages UI.

**CI Pipeline:**
- One GitHub Actions workflow: `.github/workflows/new-release.yml`.
  - Name: `new-release`
  - Trigger: `repository_dispatch` with type `new-release`
  - Permissions: `contents: write`
  - Runs on: `ubuntu-latest`
  - Steps:
    1. `actions/checkout@v5` with `persist-credentials: true`
    2. Writes the dispatch `client_payload` JSON (piped through `jq`) to `src/data/latest-release.json`
    3. Merges the new payload into `src/data/releases.json` (prepends newest, dedupes by `.version`)
    4. Commits and pushes via `stefanzweifel/git-auto-commit-action@v7` with message `chore(release): <tag>` targeting only the two JSON files.
  - Purpose: lets an external publisher (the Covalence app release pipeline) push new release metadata into this repo via `repository_dispatch`, which then triggers a new Pages build.
- No other GitHub Actions (no lint, test, preview, or deploy workflow is committed; Pages deploys directly from the `main` branch on push).

## Environment Configuration

**Required env vars:** None. The site has no runtime env var consumers, and `grep` finds no `process.env.*` or `import.meta.env.*` references in `src/` beyond what Astro itself provides internally.

**Secrets location:** No secrets are needed or stored in this repo.
- `.env` and `.env.production` are listed in `.gitignore` as a precaution, but no such files exist.
- The `new-release` workflow relies only on the default `GITHUB_TOKEN` (implicit via `persist-credentials: true`) — no repository secrets are referenced.
- Any CF Pages secrets live in the Cloudflare dashboard, not in this repo.

## Webhooks & Callbacks

**Incoming:**
- GitHub `repository_dispatch` event of type `new-release` → handled by `.github/workflows/new-release.yml`. The expected `client_payload` shape (inferred from the seeded `src/data/latest-release.json`) is:
  - `version` (string, e.g. `"1.3.2"`)
  - `tag` (string, e.g. `"v1.3.2"`)
  - `released_at` (ISO-8601 string)
  - `download_url` (string)
  - `download_size_bytes` (number)
  - `sha256` (string)
  - `notes_markdown` (string)
  - `delta_urls` (array)

**Outgoing:** None. The site does not call any webhooks or external APIs at build time or runtime.

## Third-Party Scripts & Embeds

- None. `src/layouts/Base.astro` loads only self-hosted fonts and a favicon. Starlight's generated pages include its own bundled search (`pagefind`) served from `/pagefind/` on the same origin — not an external service.

## Fonts & Static Assets

- Self-hosted fonts under `public/fonts/`:
  - `inter-latin-400.woff2`, `inter-latin-600.woff2`, `inter-latin-700.woff2`
  - `jetbrains-mono-latin-400.woff2`
- Declared via `@font-face` rules in `src/styles/global.css` with `font-display: swap`. Preloaded (400 + 600) in `src/layouts/Base.astro`.
- No Google Fonts, no Typekit, no external font CDN.

## Search

- Pagefind static index, generated by Starlight into `dist/pagefind/` at build time. Same-origin, no external search service.

## Summary

This repository ships a fully self-contained static site. The only external surface is:
1. An inbound `repository_dispatch` webhook that updates two JSON files.
2. Outbound hyperlinks to `/releases/*` on the same domain, which Cloudflare Pages forwards to a separate binary-hosting origin via `public/_routes.json`.

Everything else — auth, analytics, error tracking, databases, third-party SDKs — is intentionally absent.
